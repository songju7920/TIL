- 정의
  > **키**와 **값**으로 이루어진 저장과 조회라는 가장 간단한 원칙에 충실한 데이터베이스이다.
  **Key**와 **Value**이 **하나의 묶음으로 저장되는 구조**로, 단순하고 속도가 빠르고 분산 저장시 유용한 데이터의 형태를 가지고 있다.
- 종류
  - Redis
  - AWS DynamoDB
  - Riak
  - Oracle Barkely DB
- 구조 예시
  | Key | Value             |
  | --- | ----------------- |
  | K1  | AA, BB, CC        |
  | K2  | AA, DD            |
  | K3  | AA, 5623          |
  | K4  | AA, 2, 01/01/2023 |
